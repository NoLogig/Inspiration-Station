Never exceed the boundaries!

The JS code will load the images in the background in order to retrieve the image height, meaning the scrolling never exceeds the image boundaries, giving you a perfectly seamless parallax solution.

To ensure you make the most of your parallax images, remember to add the data-position="top" to any elements that are flush with the top of the window and data-position="bottom" to any elements flush with the bottom of the window. The script won't break if you forget to add these, but you might be wasting some valuable pixels ;)

Yep, even nests elements can be controlled from the script!

Usage:

The code looks for any elements with the following attributes:

  • data-type="parallax" - This is the only required attribute. It tells the code to add the parallax control to this element.

  • data-scroll-height="int(px)|percent" (optional) - Specify the scroll-height if you want this to differ from the actual image height. Useful for experimenting with tiled backgrounds. Defaults to background image height.

  • data-invert="true|false" (optional) - Invert the movement direction. Defaults to false.

  • data-bg-height="int(px)/percentage(%)" (optional) - Set the background height. Using this will disable loading the image in the background and scale the image to this height instead. Widths are set to auto. If this is set to a value less-than the elements height, then it will be scaled to the elements height and no parallax effect will take place.

This script can lag on some browsers/operating systems. Suggestions for code improvements are welcomed.
